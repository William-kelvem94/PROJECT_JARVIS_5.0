## 🏛️ Arquitetura de Elite

### 🧠 Câmbio Cognitivo (Dynamic RAM Management)
Diferente de sistemas estáticos, o JARVIS utiliza um gerenciamento dinâmico de memória que alterna entre modelos de linguagem (Llama 3, Qwen, Gemini) baseando-se na complexidade da tarefa e disponibilidade de hardware. Se você tem GPU, ele voa; se não, ele otimiza o uso de CPU para manter a latência sob controle.

### 🛡️ Jaula de Vidro (HITL & Action Validator)
Segurança em primeiro lugar. O JARVIS opera sob o protocolo **Human-In-The-Loop (HITL)**. Ações críticas (e.g. deletar arquivos, executar comandos de sistema) passam por um validador semântico e exigem autorização por voz ou interface antes de serem seladas.

### 📚 RAG Hierárquico (Neural Vault)
Utilizamos um sistema de Recuperação Aumentada por Geração (RAG) em camadas:
1.  **Cache de Curto Prazo**: Respostas instantâneas.
2.  **Grafo Semântico**: Conceitos aprendidos e relações rápidas em RAM.
3.  **Memória de Longo Prazo (ChromaDB)**: Bilhões de vetores que guardam cada interação e conhecimento técnico acumulado.

## 🚀 Como Iniciar

### Pré-requisitos
- Python 3.10+
- NVIDIA GPU (Opcional, mas recomendado para modo ULTRA)
- Ollama instalado

### Setup Rápido
1.  Execute o instalador de dependências:
    ```powershell
    .\INSTALL_JARVIS.bat
    ```
2.  Configure suas chaves no arquivo `.env` (opcional para modelos externos).
3.  Inicie a Singularidade:
    ```powershell
    .\START_JARVIS.bat
    ```

### Interface de Controle
- **HUD Transparente**: Feedback visual minimalista sobre o estado do Agente.
- **Dashboard Stark**: Visualização em tempo real de hardware, telemetria e logs neurais.

## 🛡️ Segurança e Resiliência
- **Blindagem de UI**: Proteção contra falhas em componentes gráficos (FallbackHUD).
- **Escrita Atômica**: Locks de thread garantem integridade total do banco de dados durante operações concorrentes.
- **Safe Mode**: O JARVIS detecta dependências corrompidas e inicia em modo de segurança se necessário.
