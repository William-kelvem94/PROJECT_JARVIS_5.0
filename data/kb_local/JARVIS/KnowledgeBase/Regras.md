# PROJECT_JARVIS_5.0 Knowledge Base Rules

Estas são as regras para manter a base de conhecimento do Jarvis organizada, consistente e expansível.

## 1. Propósito da pasta
- Esta pasta é a consciência do Jarvis e o seu segundo cérebro.
- Não é o código do projeto; o código fica em `C:\Users\willi\Documents\GitHub\PROJECT_JARVIS_5.0`.
- Use esta pasta para conceitos, regras, persona, arquitetura e saber do assistente.
- Mantenha `PROJECT_JARVIS_5.0-KnowledgeBase` como fonte canônica de verdade.

## 2. Organização de arquivos
- Use nomes consistentes: `PROJECT_JARVIS_5.0-<tema>.md`.
- Cada arquivo deve ter um título claro e frontmatter com `title`, `description` e `tags`.
- Evite duplicar conteúdo dentro da mesma pasta.

## 3. Criação de novas notas
- Antes de criar, verifique se já não existe um tema parecido.
- Use palavras-chave no nome e no título.
- Adicione o novo arquivo a [[INDEX|INDEX.md]].

## 4. Atualização de notas existentes
- Atualize as notas sempre que o Jarvis aprender algo novo.
- Se a nota mudar muito, mantenha um histórico simples no final do arquivo.
- Sincronize mudanças relevantes com `Projetos/EstudosFocados/PROJECT_JARVIS_5.0.md` se for estratégia.

## 5. Sincronização com o código
- `PROJECT_JARVIS_5.0-KnowledgeBase` = conhecimento/consciência.
- `C:\Users\willi\Documents\GitHub\PROJECT_JARVIS_5.0` = código e execução.
- Atualizações de arquitetura ou requisitos no código devem refletir na KB.

## 6. Uso pelo Jarvis
- Jarvis deve carregar `JARVIS_KB_PATH` para acessar esta pasta.
- Jarvis deve entender que este é o seu segundo cérebro e estudar esses arquivos antes de executar ações.

## 7. Exemplos de novos arquivos
- `PROJECT_JARVIS_5.0-Tools.md`
- `PROJECT_JARVIS_5.0-UseCases.md`
- `PROJECT_JARVIS_5.0-DialogFlow.md`
- `PROJECT_JARVIS_5.0-UserProfile.md`

## 8. Regras de estilo
- Use linguagem clara e direta.
- Separe conceitos em seções.
- Use listas para ações e critérios.
- Adicione referências para outros documentos.
- Sempre vincule esta nota a [[README|README.md]] e [[INDEX|INDEX.md]] para garantir visibilidade.

**Meta**: Fazer desta pasta o centro de aprendizado e expansão do Jarvis.
